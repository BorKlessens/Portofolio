Media Development - EnoRm & Fightclub

De opdracht: (Media Development)

Ontwikkel een interactieve easter egg die de overgang van EnoRm naar Fightclub markeert en op speelse wijze de identiteit van beide bedrijven samenbrengt. Denk aan verborgen animaties, geheime interacties of verrassende visuele transformaties die de bezoeker ontdekken.

Proces

Voor de start van dit project kregen we eerst een presentatie over de geschiedenis van Fightclub en EnoRm. Hierin werd uitgelegd waar deze twee bedrijven voor stonden om de opdracht voor ons duidelijker te maken. Nadat de opdracht duidelijk was, konden we aan de slag.

JavaScript Leren

Omdat ik nog geen ervaring had met JavaScript, ben ik eerst veel onderzoek gaan doen. Ik heb hiervoor verschillende filmpjes bekeken over de basics van JavaScript. Ook heb ik verschillende kleine projectjes gemaakt die eigenlijk niks met de opdracht te maken hadden. @oefen project Stop Watch @oefen project - color change Daarnaast heb ik om te oefenen de opdracht van David gemaakt: @Oefen Opdracht David Met deze kleine projectjes heb ik de basis van JavaScript geleerd. Dit was ook mijn belangrijkste doel voor deze opdracht.

Easter Eggs Ontwikkeling

Toen ik wat meer over JavaScript had geleerd, begon ik met het bedenken van ideeën voor de easter eggs. Deze ideeën heb ik samen met Thijn op papier uitgewerkt om ze beter te visualiseren. We hebben kleine tekeningen met beschrijvingen gemaakt om het idee duidelijker te maken. Ook hebben we alle benodigde items erbij gezet en meteen gemaakt. @Uitgewerkte ideeën easter eggs

Website en Logo Animatie

De eerste easter egg die ik heb gemaakt, was bij het logo. Het idee was dat wanneer je op het logo klikt, het langzaam zou vervagen en overgaat in het logo van EnoRm. Eerst wilde ik dit doen met een @gif logo glitch , maar ik kwam er al snel achter dat VS Code en gifjes niet goed samenwerkten. Daarom heb ik de animatie verder uitgewerkt met CSS en JavaScript. @logo flikker

Code Organisatie

Volledige Website

Gedurende het project hield ik me ook bezig met het beheren van de website. Ik heb eerst zelf een website gebouwd met HTML en CSS en er vervolgens verschillende JavaScript-elementen aan toegevoegd om alle easter eggs samen te voegen. Dit duurde langer dan verwacht, omdat ik veel tijd kwijt was aan het opschonen van de code en het oplossen van fouten die ontstonden toen alles werd samengevoegd. De easter eggs werkten afzonderlijk prima, maar eenmaal samengevoegd gingen ze met elkaar botsen. Daarom besloot ik alles over te zetten naar een nieuwe opzet met een screenshot van de Fightclub-website als achtergrond. Uiteindelijk heb ik alle fouten kunnen oplossen en de code netjes geordend, zodat alles overzichtelijk en makkelijk terug te vinden was. @code netjes geordend

GitHub en Vercel

Ik heb samen met Thijn de code gedeeld in een respository in github. Hiermee konden we de aanpassingen telkens met elkaar delen zodat we niet twee aparte projecten gingen creëren. Ook staat ons proces op deze manier goed gedocumenteerd en kunnen de docenten gemakkelijk de code bekijken om het leerdoel Development te beoordelen. Daarnaast hebben we de website gehost op een vercel zodat hij door iedereen bekeken kan worden. https://github.com/Thijn-Opperman/Semester-2---Fightclub---Easter-egg. https://semester-2-fightclub-easter-egg-rk8c.vercel.app/

Reflectie

Door deze opdracht heb ik veel geleerd over JavaScript. Aan het begin wist ik er nog niets van, dus het was even wennen. Maar zodra ik de basis doorhad, begon het leuker te worden en kreeg ik steeds beter resultaat. Al met al was het een leerzame en leuke opdracht die goed aansluit bij mijn interesse in Front End Development.
